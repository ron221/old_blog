前言

想要用 Hexo 自己搭一個部落格完全是心血來潮，花了一天的時間終於完成了，還滿有成就感的！原本就有寫部落格的習慣，一開始在 Wordpress、後來被 Medium 簡潔的介面吸引過去，直到現在嘗試用 Hexo + Github 來搭建一個屬於自己的部落格。

Hexo 是一個靜態的部落格生成器，能把我們使用 Markdown 語法寫好的文章生成方便瀏覽的網頁，同時提供了部署的功能，因此我們可以方便地將它部署到 Github 上。而 Github 這裡當作靜態檔案的伺服器，託管由 Hexo 生成的網頁。

Hexo建站

首先要安裝 Hexo，參考 Hexo 文件，先安裝好 node.js 和 Git ，然後在 terminal 直接安裝即可：

接著：

  1. 任何地方建立一個 your_name.github.io 的資料夾
  2. 在 terminal 用 cd 進到該資料夾
  3. terminal 執行 hexo init ，初始化部落格的環境
  4. terminal 執行 npm install 安裝好環境
  5. 這時在 terinal 執行 hexo server，就可以在本地 local:4000 看到網站了

設定網域名

因為是心血來潮想建部落格，所以連網域名也順便買了一個！我是在 Godaddy 上買的，還有 Gandi、阿里云很多其他的網站上可以買，甚至有比價網站可以先查一下再買。花了兩年兩萬韓元的價格，台幣約 500 塊，其實還有很多更便宜的，但想說還是不要買太奇怪、太長的名字，於是買了個 .me 的。

  1. 在 source 的文件夾裡新增一個叫 CNAME 的檔案，裡面指加上一行網域名即可： ronyeh.me。用來告訴 Github your_name.github.io是對應到哪個網域名
  2. 生成內容、部署到 Github
  1. 最後在 Godaddy 網站上完成設定即可

總結

整個搭建的過程不會太複雜，只是有時候遇到 error 就卡個半天，像我在安裝 hexo 的時候一直有權限的問題，無法成功安裝。不過主要的搭建過程都算容易，另外文中沒有提到更換主題，我使用的模板是: hexo-theme-beantech，簡潔好看、客製化設定容易。寫 Markdown 文章我是使用 Typora，免費、使用起來也方便。但覺得 Markdown 語法寫的習慣後，可以直接用 Atom 編輯。

希望之後在寫文章的同時也能有更多新發現啦！
